Zorvyn FinTech Full Stack Project Summary (Current State)
Date: 2026-04-02

1) Project Purpose
This project is a full-stack personal/organizational finance dashboard platform with:
- A secured backend API (Node.js + Express + MongoDB/Mongoose)
- A role-aware frontend web app (React + Vite + Tailwind)

The platform supports:
- Registration and login using JWT-based authentication
- Role-based access control (VIEWER, ANALYST, ADMIN)
- Financial record CRUD with validation, filters, search, pagination, and soft delete
- Dashboard analytics (summary, category breakdown, recent activity, monthly trends)
- Frontend reliability UX (loading, errors, retry, empty states, request cancellation)
- UI safety helpers (toasts, modal forms, error boundary)
- Seed data and automated tests for backend and frontend core behavior


2) High-Level Architecture
Backend:
- Express 5 API with modular MVC-style organization
- Controllers for request handling, service layer for analytics
- Mongoose models with schema validation and query indexes
- Middleware-based auth, authorization, rate limiting, validation, and global errors

Frontend:
- React SPA with route-level protection and role gates
- Context-based auth and toast state
- Central axios instance with interceptors for auth headers and 401 handling
- Domain API modules for auth, records, and dashboard
- Reusable UI components (layout, modal, form, chart, toast, boundary)

5) Backend Detailed Implementation

5.1) Server and App Bootstrap
- src/server.js:
  - Loads dotenv
  - Connects DB before starting HTTP listener
  - Handles unhandled promise rejections
  - Exports startServer for controlled test/runtime usage

- src/app.js:
  - CORS middleware with configurable allowed origins
  - JSON body parsing
  - Root welcome endpoint GET /
  - Route mounting for auth, users, records, health, dashboard
  - Global notFound and error middleware

5.2) CORS Behavior
- Enabled via cors package in src/app.js
- Defaults allow:
  - http://localhost:5173
  - http://127.0.0.1:5173
- Can be overridden with CORS_ORIGIN in backend env
- Handles browser preflight for frontend API requests

5.3) Database and Models
- src/config/db.js:
  - DB connect function with env checks and fail-fast startup behavior

- src/models/User.js:
  - Fields: name, email, password, role, status, timestamps
  - Email validation and uniqueness
  - Password hidden by default in queries (select: false)
  - Pre-save bcrypt hashing
  - comparePassword helper
  - Role enum: VIEWER, ANALYST, ADMIN
  - Status enum: ACTIVE, INACTIVE

- src/models/FinancialRecord.js:
  - Fields: userId, amount, type, category, date, note, isDeleted, timestamps
  - amount > 0 validation
  - type enum INCOME/EXPENSE
  - Soft delete using isDeleted
  - Indexing for type/category/date/isDeleted query paths

5.4) Middleware
- src/middleware/authMiddleware.js:
  - protect: validates Bearer token, verifies JWT, checks user existence/status
  - authorizeRoles(...roles): role gate for protected routes

- src/middleware/authRateLimiter.js:
  - 15-minute window
  - max 10 requests per IP
  - Applied to register and login routes

- src/middleware/financialRecordValidation.js:
  - create payload validation and normalization
  - update payload partial validation

- src/middleware/errorHandler.js:
  - notFound middleware for unmatched routes
  - global error response with safe stack handling by NODE_ENV

5.5) Controllers and Services
- src/controllers/authController.js:
  - register:
    - validates name/email/password
    - checks duplicate email
    - accepts optional role if in allowed enum
    - returns created user payload (no password)
  - login:
    - validates input
    - checks credentials and ACTIVE status
    - returns JWT token + user payload

- src/controllers/userController.js:
  - admin endpoint to update user status

- src/controllers/financialRecordController.js:
  - create/get list/get by id/update/delete (soft delete)
  - list supports:
    - filter: type, category, startDate, endDate
    - search: category + note (escaped regex)
    - pagination: page, limit, totalCount, totalPages
  - request validation and defensive error outputs

- src/controllers/dashboardController.js:
  - thin controller delegating analytics logic to service layer

- src/services/dashboardService.js:
  - getSummary -> totalIncome, totalExpense, netBalance
  - getCategoryBreakdown -> grouped totals by category with income/expense split
  - getRecentActivity -> latest records with limit 5 or 10
  - getMonthlyTrends -> month-wise income and expense aggregates
  - shared date/type/category filter builder with strict validation

5.6) Routes and Access Matrix
- src/routes/authRoutes.js:
  - POST /auth/register (rate limited)
  - POST /auth/login (rate limited)

- src/routes/userRoutes.js:
  - PATCH /api/users/:id/status (ADMIN only)

- src/routes/recordRoutes.js:
  - GET /records (ANALYST, ADMIN)
  - GET /records/:id (ANALYST, ADMIN)
  - POST /records (ADMIN)
  - PATCH /records/:id (ADMIN)
  - DELETE /records/:id (ADMIN, soft delete)

- src/routes/dashboardRoutes.js:
  - Protected for VIEWER, ANALYST, ADMIN
  - GET /summary
  - GET /category-breakdown
  - GET /recent
  - GET /trends
  - Aliases:
    - GET /recent-activity
    - GET /monthly-trends
  - Mounted at both /dashboard and /api/dashboard

- src/routes/healthRoutes.js:
  - GET /api/health

5.7) Token Utility
- src/utils/generateToken.js:
  - signs JWT with user id + role
  - default expiry 1 day

5.8) Seed Data
- src/scripts/seed.js:
  - clears existing users and records
  - creates 3 demo users:
    - admin@demo.com / Admin123 (ADMIN)
    - analyst@demo.com / Analyst123 (ANALYST)
    - viewer@demo.com / Viewer123 (VIEWER)
  - inserts 10 demo records tied to admin user
  - logs seed summary and closes DB connection

5.9) Backend Test Coverage
- tests/auth.test.js (Jest + supertest):

7) End-to-End Security and RBAC Behavior
Backend enforcement (source of truth):
- protect middleware validates token and active user status
- authorizeRoles enforces route-level role permissions

Frontend enforcement (UX layer):
- ProtectedRoute blocks unauthenticated access
- /records is role-gated (ADMIN/ANALYST)
- AppLayout hides Records nav for VIEWER
- axios interceptor handles expired/invalid sessions via 401 auto-logout redirect

Role matrix in current behavior:
- VIEWER:
  - dashboard allowed
  - records route blocked
- ANALYST:
  - dashboard allowed
  - records read allowed
  - create/update/delete blocked by route/UI/backend
- ADMIN:
  - full dashboard + records access
  - create/update/delete records allowed
  - can update user status via backend user route


8) Reliability and UX Patterns
- Request cancellation in dashboard and records to avoid stale state races
- User-friendly error messages with Retry controls
- Loading and empty-state rendering throughout key pages
- Toast notifications for success and failure actions
- Modal close protections during in-flight submission
- Global error boundary for unexpected render crashes
